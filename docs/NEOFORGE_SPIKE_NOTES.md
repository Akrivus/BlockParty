# NeoForge 1.21.4 Build Spike Notes

This branch is a build/load spike only. It intentionally compiles a minimal NeoForge source path from `src/neoforgeSpike` and leaves the Forge 1.19.4 production code in place for later porting.

## Active Spike Surface

- `build.gradle`: switched from ForgeGradle to NeoForge ModDevGradle for NeoForge `21.4.102-beta`, Java 21, and minimal client/server/GameTest/data runs under `run-neoforge-spike`.
- `settings.gradle`: added NeoForge and Gradle plugin repositories for ModDevGradle.
- `gradle/wrapper/gradle-wrapper.properties`: moved the wrapper to Gradle 8.8 for ModDevGradle compatibility.
- `src/neoforgeSpike/java/block_party/BlockParty.java`: minimal `@Mod("block_party")` entrypoint preserving mod id and package name.
- `src/neoforgeSpike/java/block_party/gametest/SpikeGameTests.java`: one empty required GameTest using the existing `block_party:empty` template name.
- `src/neoforgeSpike/java/block_party/gametest/RegistryGameTests.java`: representative NeoForge registry ID smoke test for block, item, sound, particle, entity type, scene action, and scene filter registration.
- `src/neoforgeSpike/java/block_party/gametest/ResourceGameTests.java`: representative resource smoke tests for converted tags, bundled JSON reload parsing, block alias reload construction, name/alias/sound/texture metadata parsing, override-shaped resource JSON, malformed optional metadata, and Moe texture fallback paths.
- `src/neoforgeSpike/java/block_party/gametest/PersistenceGameTests.java`: persistence smoke tests for SavedData serialization, `DimensionDataStorage` caching, SQLite open/close, `HidingSpots`, and `SceneVariables`.
- `src/neoforgeSpike/java/block_party/gametest/EntityDataGameTests.java`: entity data smoke tests for spawning `moe`, stable entity IDs, Moe/MoeInHiding save/load round-trips, Moe attributes, combat delegation, home helpers, inventory NBT, inventory-derived slouch, block aliases, volume scale, and Layer7 timer NBT.
- `src/neoforgeSpike/java/block_party/gametest/MoeLifecycleGameTests.java`: lifecycle smoke tests for tagged block spawn, invalid spawn no-op, source block removal, survival/creative spawn egg consumption, owner/source-state/block-entity data persistence, owner-list insertion/de-duplication, hide, HidingSpots recording, manual/timed/event reveal, block-entity data hide/reveal round-trip, and missing hidden spot no-op behavior.
- `src/neoforgeSpike/java/block_party/gametest/NpcServiceGameTests.java`: server-side companion record service tests for owner lists, owned row access, de-listing, safe rejection, and hide/reveal list stability.
- `src/neoforgeSpike/java/block_party/gametest/NetworkPayloadGameTests.java`: NeoForge custom payload codec and server-handler tests for NPC list/detail/remove/call scaffolding, controller-open payload state, dialogue open/respond/close payloads, and shrine list sync.
- `src/neoforgeSpike/java/block_party/gametest/CellPhoneServiceGameTests.java`: server-side Cell Phone call service tests for visible owned Moe teleport, far loaded Moe calls, ownership rejection, safe failures, hidden rejection, unloaded rejection, forced chunk cleanup, and following flag behavior.
- `src/neoforgeSpike/java/block_party/gametest/BlockEntityGameTests.java`: server-side block entity parity tests for legacy block entity IDs, block placement shell creation, data block NBT round-trip, SQLite table creation, locative row fields, row deletion, and Forge owner-or-dimension shrine list filtering.
- `src/neoforgeSpike/java/block_party/network/CustomMessenger.java` and `src/neoforgeSpike/java/block_party/network/payload/**`: typed NeoForge custom payload registration for NPC list/detail/remove/call requests, list/detail/call responses, no-op clientbound controller-open state, dialogue open/respond/close contracts, and shrine list request/response sync.
- `src/neoforgeSpike/java/block_party/client/**`: first NeoForge client render path for Moe and MoeInHiding, including the `block_party:moe` model layer, render-state-backed Moe model, renderer registration, name/health label, texture fallback, and emote/glow/special layer shells.
- `src/neoforgeSpike/java/block_party/world/chunk/ForcedChunk.java`: minimal server-side forced chunk tracker for Cell Phone row lookup and cleanup parity.
- `src/neoforgeSpike/java/block_party/blocks/AbstractDataBlock.java` and `src/neoforgeSpike/java/block_party/blocks/entity/**`: minimal server-side data block and block entity shells for garden lanterns, hanging scrolls, paper lanterns, sakura saplings, shimenawa, shrine tablets, and wind chimes.
- `src/neoforgeSpike/java/block_party/entities/Moe.java`: thin NeoForge entity shell preserving database ID, owner UUID, source block state, copied source block-entity persistent data, following flag, and lightweight profile strings.
- `src/neoforgeSpike/java/block_party/entities/MoeInHiding.java`: thin NeoForge entity shell preserving database ID, attached block position, `HideUntil`, and `ticksHidden`.
- `src/neoforgeSpike/java/block_party/items/CustomSpawnEggItem.java`: minimal `moe_spawn_egg` behavior for blocks tagged `block_party:spawns_moes`.
- `src/neoforgeSpike/java/block_party/registry/CustomResources.java`: minimal server reload listener wiring for inert bundled JSON parsing.
- `src/neoforgeSpike/java/block_party/registry/resources/CountingJsonReloadListener.java`: narrow parser/counting reload listener for `moes/aliases`, `moes/names`, `moes/sounds`, and scene resource smoke counts.
- `src/neoforgeSpike/java/block_party/registry/resources/ScenesReloadListener.java`: server reload listener for real scene parsing/execution data under `data/*/scenes`, including trigger grouping and bundled scene lookup.
- `src/neoforgeSpike/java/block_party/db/BlockPartyDB.java`: NeoForge `SavedData.Factory` scaffold preserving `BlockParty_DB`, `Names`, `NPCsByPlayer`, and the world-local `blockparty.db` SQLite path.

- `src/neoforgeSpike/java/block_party/db/records/NPC.java`: minimal SQLite-backed `NPCs` row adapter for Moe identity, owner UUID, block state, hiding state, hidden position, and lightweight profile fields.
- `src/neoforgeSpike/java/block_party/scene/SceneVariables.java` and `src/neoforgeSpike/java/block_party/scene/data/**`: NeoForge `SavedData.Factory` preserving `BlockParty_SceneVariables`, cookies, counters, locations, and targets NBT shape. Slice 5.1 connects Moe-scoped cookie/counter scene actions and minimal cookie/counter filters.
- `src/neoforgeSpike/resources/META-INF/neoforge.mods.toml`: minimal NeoForge mod metadata preserving mod id, display name, package-facing identity, and Minecraft/NeoForge dependency ranges.
- `src/neoforgeSpike/resources/data/block_party/structure/empty.nbt`: copied empty GameTest structure template for the NeoForge 1.21.4 resource path.
- `syncNeoForgeSpikeResources` in `build.gradle`: copies safe legacy assets and data into `build/generated/neoforgeSpikeResources`, and converts legacy `tags/blocks` / `tags/items` to 1.21.4 `tags/block` / `tags/item`.

## Semantic Drift Notes

- Slice 1.1 spawn item consumption deliberately differs from the frozen Forge 1.19.4 source in creative mode: Forge's item implementation shrinks the stack after a successful spawn, while the NeoForge spike now consumes in survival and preserves the stack in creative. This drift is covered by GameTests because Slice 1.1 explicitly calls for survival-vs-creative behavior.
- Slice 1.2 piston reveal checks both the piston event position and the block in front of the piston. The frozen Forge 1.19.4 handler checked only the event position, but the NeoForge spike uses the broader check to preserve the player-facing contract that piston disturbance of a hidden block reveals the Moe.
- Slice 1.3 keeps cross-dimension Cell Phone calls as safe failures. Forge uses `ITeleporter` for dimension changes, but the NeoForge spike has not ported the entity teleport/dimension-change shell yet, so this slice only restores same-dimension row-position lookup and forced chunk cleanup.
- Slice 1.3's far-call GameTest now uses an in-template distant source position instead of the old 48-block relative fixture. The 48-block position reached into neighboring GameTest space once Phase 1 added more tests; the service contract remains forced chunk queue/release plus same-dimension loaded Moe movement.
- Slice 1.4 does not execute Forge shrine tablet side effects yet. The frozen Forge shrine tablet creates lightning, spawns a Moe below the tablet, plays sound, and broadcasts `SShrineList` after row changes. The NeoForge spike currently restores server-side block entity IDs, NBT, SQLite rows, Forge-parity shrine list query behavior, and Slice 3.3 typed shrine list request/response plus login payload sends; visual/audio/entity side effects and shrine row-change broadcasts remain deferred.
- Slice 1.4 leaves `ShimenawaBlockEntity` as a registered server-side shell without creating/updating an `NPCs` row on placement. Forge uses shimenawa persistent data to back an NPC record, but that path depends on the fuller NPC row/profile port and remains a documented stub.
- Slice 2.2 supports blood type, dere, and zodiac trait tags, but the frozen bundled Forge 1.19.4 tag JSON for those categories is mostly empty. Current GameTests therefore cover non-empty baseline tags such as pronouns, wings, glow, cat features, and volume ignoring, plus fallback parsing for the string traits.
- Slice 3.3 restores the frozen Forge `SShrineList` filter exactly as `owner UUID matches OR dimension matches`. Earlier spike notes described this as owner/dimension filtering, but the active NeoForge query now intentionally permits same-dimension shrine rows owned by other players because that is what the Forge 1.19.4 packet sent.
- Slice 5.1 represents absent response text as an empty string in active NeoForge `Dialogue` payloads. Forge `SendResponse` could use translated fallback text; the spike keeps response icons and nested action behavior active while leaving richer translated response labels for a later UI/content polish slice.
- Slice 5.2 accepts Moe texture metadata from both `moes/textures` and `textures/moe`. The frozen Forge class names `moes/textures`, but the shipped branch stores note-block/cake metadata under `assets/*/textures/moe`; accepting both paths preserves shipped content while still allowing data-pack style metadata.
- Slice 5.3 migrates legacy Forge resource files only in the generated NeoForge resource output: recipes move to `data/*/recipe` with 1.21.4 result/ingredient shapes, loot tables move to `data/*/loot_table` with renamed condition/predicate fields, and worldgen `ignoreVines` becomes `ignore_vines`. Source resources stay frozen for the Forge baseline.
- Slice 5.3 keeps the old placed-feature JSON content, including the frozen branch's existing feature references, rather than retuning tree distribution or correcting authored content opportunistically.
- Slice 5.3.5 restores a NeoForge creative tab for PR review discoverability. The frozen Forge branch only had commented creative-tab code, so this is treated as a manual-review readiness port rather than new gameplay. Exact Forge sort-order helper parity remains deferred until normalization decides whether to keep or replace the old `ISortableItem` path.
